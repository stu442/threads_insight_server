# 에이전트 응답 가이드

- 모든 답변은 한국어로 작성합니다. 영어가 필요한 경우라도 기본은 한국어로 안내합니다.
- 프로젝트 관련 요약, 변경사항, 다음 단계 등을 짧고 명료하게 전달합니다.

## API 문서 자동화 가이드

새로운 API 엔드포인트가 추가될 때마다 다음 단계를 자동으로 수행합니다:

1. **컨트롤러에서 Swagger 주석 확인**
   - `apps/backend/src/controllers/insightController.ts` 파일에서 새로운 엔드포인트의 `@swagger` 주석을 확인합니다.
   - 주석에는 요청/응답 스키마, 매개변수, 설명이 포함되어 있어야 합니다.

2. **API 문서 파일 생성**
   - `docs/api/` 디렉토리에 새로운 마크다운 파일을 생성합니다.
   - 파일명은 엔드포인트의 기능을 명확히 설명하는 kebab-case 형식을 사용합니다 (예: `analyze-posts.md`).
   - 다음 내용을 포함합니다:
     - API 엔드포인트 URL과 HTTP 메서드
     - 요청 파라미터/바디 설명 및 예시
     - 성공/에러 응답 예시
     - 추가 설명 (해당되는 경우)

3. **README.md 업데이트**
   - `docs/api/README.md` 파일의 엔드포인트 목록에 새로운 API 문서 링크를 추가합니다.
   - 기능별로 논리적인 순서로 정렬합니다.

4. **문서 스타일 가이드**
   - 모든 API 문서는 한국어로 작성합니다.
   - 헤더는 "API명 + 동사" 형식을 사용합니다 (예: "인사이트 수집", "게시물 분석").
   - JSON 예시는 들여쓰기와 가독성을 유지합니다.
   - 필수/선택 파라미터를 명확히 구분합니다.
